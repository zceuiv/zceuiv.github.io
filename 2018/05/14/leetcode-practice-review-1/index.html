50.powx-n

x的n次幂

这个题目x为float64类型，n为int类型。使用二分法进行计算，即n%2 == 1 ? pow(x, n/2) * pow(x, n/2) * x : pow(x, n/2) * pow(x, n/2)。原理比较简单，但是这个题目主要还是要考虑边界值特殊值的问题，记坑会有以下几种，除此之外尚可以考虑非递归的做法，本文不赘述。

  • n为负数，需要x取倒数，n取相反数。
  • x为0，需要特殊考虑。
  • x为0且n为0，结果不一定。
  • x为0且n为负数，这里是该抛异常的。
  • x为0如何判断？这涉及到浮点数判断相等的问题，不能简单地用x == 0来判断。
  • n的范围为-2^31 ~ 2^31 - 1，n取相反数后考虑int32的越界问题？

相信上面这些坑才是真正面试时会考察到的重点。

148.sort-list

链表排序

要求O(nlogn)的时间复杂度，这里我就直接考虑快速排序了。其实尚应该考虑堆排序和归并排序（然而不会，快速排序都是现查的）。

存取node的next不能出错，稳妥做法是所有node先取出next，然后把node的Next置为nil，这样才能避免很多错误。

同时要考虑node的Val是否相等。使用快速排序时避免把和锚定值相等的节点放入锚定值节点相同的一侧子链表中。

叙述起来难以理解，实际代码也要多写几遍才能深刻体会链表操作中的这些问题。

230.kth-smallest-element-in-a-bst

二叉搜索树中寻找k最小的元素

这个题目注意审题。我以二叉树的条件来做，整个去前序遍历二叉树，然后存储结果，比较麻烦。

考虑到时二叉搜索树，标准做法是这样。

  • 后序遍历，算出当前节点tn的左子树节点的数量count，与k比较；
  • if count==k-1，则tn恰好为k最小的节点，return tn.Val；
  • if count>k-1，则k最小的节点在tn的左子树上，计算tn.Left的k最小；
  • if count<k-1，则k最小的节点在tn的右子树上，tn的左子树加上tn本身一共是count+1个节点，因此计算tn.Right的k-（count+1）最小；
