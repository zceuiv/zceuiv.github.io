2.3 服务

准备工作

  • Docker1.13或以上版本。
  • Docker Compose。Docker for Mac和Docker for Windows已经预安装。
  • 学习了教程1和教程2。
  • 确保公开了教程2中的friendlyhello镜像。
  • 确定你的镜像能够作为一个容器工作。先运行docker run -p 80:80 username/repo:tag然后访问http://localhost/。

介绍

关于服务

分布式应用程序中，程序中不同的部分被称为服务。例如，如果做一个视频分享网站，可能会划分存储为一个服务，后台上传视频并转码为一个服务，前端为一个服务，等等。

Services其实就是“containers in production.”一个服务只运行一个镜像，但是它规范容器运行的方式：应该用哪个端口，运行多少份容器的复制，等等。扩展一个服务改变容器实例运行的数量、分配计算资源给那个服务。

你的第一个docker-compose.yml文件

这个docker-compose.yml文件告诉Docker做如下事情：

  • 拉取镜像。
  • 运行那个镜像的5个实例作为一个服务，命名为web，限制每一个实例最多10%的CPU(按所有核心算)和50MB内存。
  • 当某个容器宕掉时立刻重启它。
  • 映射主机80端口到web服务的80端口。
  • 使所有web服务的容器共享80端口，通过一个负载均衡的网络，命名为webnet。
  • 使用默认设置定义webnet网络(负载均衡的覆盖网络)。
