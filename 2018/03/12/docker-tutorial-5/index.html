2.5 Stack

准备工作

  • Docker1.13或以上版本。
  • Docker Compose。Docker Machine。
  • 学习了教程1和教程2。
  • 确保公开了教程2中的friendlyhello镜像。
  • 确定你的镜像能够作为一个容器工作。先运行docker run -p 80:80 username/repo:tag然后访问http://localhost/。
  • 教程3里的docker-compose.yml文件的拷贝。
  • 教程4中准备的docker-machine可用。docker-machine ls，docker-machine start myvm1。
  • 教程4中的swarm可用。docker-machine ssh myvm1 "docker node ls"

简介

教程4中学习了启动swarm作为Docker集群，并且部署了一个应用程序到多机器上同步运行。
教程5介绍stack。stack是一组相互关联的服务，它们共享依赖项，可以共同编排和扩展。一个单独的stack是能够定义和协调一整个应用的功能的(当然特别复杂的应用们会需要使用多个stack)。
这里你会部署多个服务到多个机器上。

添加一个新的服务并重新部署

  1. 打开docker-compose.yml文件，做如下修改。

这里新增了一个和web同等地位的服务，名为visualizer。两个新的东西：volumes关键字，试的visualizer访问主机的socket文件；placement关键字，确保这个服务只运行在swarm管理机上。这是因为visualizer这个容器是用来图形化显示Docker服务如何放在swarm。

  1. 确保你的shell配置为和myvm1对话
  • 运行docker-machine ls列出虚拟机列表
  • 如果需要，重新运行docker-machine env myvm1，然后eval $(docker-machine env myvm1)。

  1. 重新在管理机上运行docker stack deploy命令，这样需要更新的服务就会更新。

  2. 查看visualizer
    compose文件中看到visualizer运行在8080端口，找到docker-machine的任意一个ip的8080端口访问visualizer。

数据持久化

同样的流程加入一个Redis数据库用来存储数据。

  1. 存储新的docker-compose.yml文件

几个呃呃呃重要的有关redis的地方说一下，有点特殊。

  • redis总是跑在manager上，所以它们总是使用相同的文件系统。
  • redis访问主机文件系统上任意一个都可以的目录映射到容器中的/data目录，这里用来存储数据。

这两点使得Redis数据存储在物理主机文件系统真实源上。不这样做，当容器重新部署时会抹掉存储的数据。

  1. 在管理机上创建./data目录

  2. 确保shell配置为和myvm1对话

  3. 再次运行docker stack deploy

  4. 运行docker service ls确认3个服务都如期运行

  5. 访问http://192.168.99.101查看访问计数，这个计数存储在Redis中。
    同样的，检查8080端口中的visualizer，看到redis服务运行
